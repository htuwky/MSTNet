ffmpeg -i video/video.mp4 -f image2 frames/frame_%05d.png
check_alignment,py脚本作用
1. 摸底视频长度 (Video Baseline)动作：它扫描 dataset/Frames 文件夹，读取所有的 frame_xxxxx.png。目的：找出 最大帧号（例如 7891）。计算：结合 config.VIDEO_FPS (23.0)，它能算出视频的真实物理时长（例如 343.09秒）。这是判定一切的基准线。2. 模拟映射测试 (Simulation)动作：它遍历 dataset/CSV 下的每一个被测者文件。计算：读取 CSV 中的最后一行时间戳 (Timestamp)，利用公式 帧号 = int(时间戳 * FPS) + 1，计算出该被测者看完整个流程所需要的那一张图片的帧号。比如：如果时间戳是 342.98秒，FPS是23，它算出你需要第 7889 号帧。3. 越界碰撞检测 (Boundary Check)核心逻辑：它将“需要的帧号”与“实际拥有的最大帧号”进行对比。情况 A (✅ OK)：需要的帧号 $\le$ 实际最大帧号。说明视频够长，数据安全。情况 B (⚠️ OOB)：需要的帧号 $>$ 实际最大帧号。说明眼动数据比视频还长（比如视频播完了，眼动仪还在录）。💡 它的价值如果不跑这个脚本，直接去跑特征提取：一旦遇到“情况 B”，程序跑了几个小时后，会在最后几秒钟报错 FileNotFoundError（因为找不到那一帧图片），导致前功尽弃。现在的作用：它提前告诉你是否需要开启“截断逻辑”。对于您的项目现状：根据您之前的运行结果，这个脚本显示 ✅ Perfect Alignment! No truncation needed.。这意味着您的数据质量极高，所有眼动数据都老老实实地落在了视频时长范围内，您可以放心地运行下一步的特征提取。
